## 📋 PRD 撰寫標準檢查清單

每個 PRD 必須包含：
- [ ] FR-ID 唯一識別碼（格式：FR-[主模組]-[子模組]-[序號]）
- [ ] 功能描述（條件/觸發、行為、資料輸入/輸出）
- [ ] UI 反應說明
- [ ] 例外處理機制
- [ ] 優先級設定（P0/P1/P2）
- [ ] 驗收標準（YAML 格式）
- [ ] API 契約定義
- [ ] 資料模型（TypeScript interface）
- [ ] 流程圖（Mermaid 語法）
- [ ] 測試案例對應

## 🚀 執行策略

### 分工原則
1. **按專業分組**：前端、後端、資料庫專家各司其職
2. **模組耦合考量**：相關模組由同一人負責
3. **平行作業**：不同模組組可同時進行
4. **每日同步**：站會追蹤進度，及時解決阻礙

### 品質保證
1. **Peer Review**：每個 PRD 需經過交叉審查
2. **一致性檢查**：使用自動化腳本驗證格式
3. **完整性驗證**：確保所有必填欄位都已填寫
4. **可測試性**：驗收標準必須可量化、可測試

### 進度追蹤
1. **每日更新**：在此表格更新狀態
2. **週報總結**：每週五提供進度報告
3. **風險預警**：提前識別可能延遲的項目
4. **資源調配**：根據進度動態調整人力
